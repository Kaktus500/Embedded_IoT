Enumeration Type Documentation

◆ timer32divider

enum timer32divider

specifies the prescale used for the 32-bit timer

Enumerator
T32DIV1 

divide by 1, maximum period of about 89 seconds at 48 MHz

T32DIV16 

divide by 16, maximum period of about 23 minutes at 48 MHz

T32DIV256 

divide by 256, maximum period of about 6 hours, 21 minutes at 48 MHz

◆ ADC_In17_14_16()

void ADC_In17_14_16 ( uint32_t *  ch17,
uint32_t *  ch14,
uint32_t *  ch16 
)

Trigger ADC measurement on channels 17+14+16 and wait for result.

Trigger a single ADC measurement on P9.0/A17, P6.1/A14, and P9.1/A16, wait for it to complete, and put the results in the pointers given. The pointers are 32-bit integers because the ADC registers are 32 bits. In this case, bits 31-16 are undefined, and bits 15-14 are zero. Busy-wait synchronization used. The ADC input voltage range is 0 to 3.3V. Uses return by reference parameter passing.

Parameters
ch17is a pointer to store 32-bit P9.0/A17 conversion result
ch14is a pointer to store 32-bit P6.1/A14 conversion result
ch16is a pointer to store 32-bit P9.1/A16 conversion result
Returns
none
Note
Assumes ADC0_InitSWTriggerCh17_14_16() has been called.

◆ Clock_Delay1ms()

void Clock_Delay1ms ( uint32_t  n)

Software implementation of a busy-wait delay.

Simple delay function which delays about n milliseconds. It is implemented with a nested for-loop and is very approximate.

Parameters
nis the number of msec to wait
Returns
none
Note
This function assumes a 48 MHz clock. This implementation is not very accurate. To improve accuracy, you could tune this function by adjusting the constant within the implementation found in the Clock.c file. For a more accurate time delay, you could use the SysTick module.
