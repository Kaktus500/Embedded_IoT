\doxysubsection{Detailed Description}
Fuzzy Logic functions. 

{\bfseries{Fuzzy logic}} is a design pattern for digital controllers. The key principle is true/false conditions can take on a spectrum of possibilities from definitely true (255) to definitely false (0). For example, a fuzzy logic value of 128 means maybe. Like any controller, a fuzzy logic controller\+: collects information from input sensors, performs calculations, makes decisions, and affects the system by outputting to actuators. Also similar to other controllers, the goal is to drive the system into a desired state. In a fuzzy logic controller, {\bfseries{crisp inputs}} are the sensor measurements with units, and {\bfseries{crisp outputs}} are the actuator commands with units. A {\bfseries{fuzzy membership set}} is a collection of variables, having values from~\newline
 true (255,254,253,...,2,1,0) false.~\newline
 The {\bfseries{input membership set}} defines the status of the system, and the fuzzy values are derived from the sensors. The {\bfseries{output membership set}} defines the status of the actuators, and the values describe what the controller intends to do in order to drive the system into the desired state. More specifically, a fuzzy logic performs these steps\+:~\newline
 1) {\bfseries{Measurements}}\+: collect {\bfseries{crisp inputs}} from sensors. ~\newline
 2) {\bfseries{Fuzzification}}\+: converts crisp inputs to input fuzzy variables.~\newline
 3) {\bfseries{Fuzzy logic}}\+: converts input fuzzy variables to output fuzzy variables. ~\newline
 4) {\bfseries{Defuzzification}}\+: converts output fuzzy variables to actuator commands, and perform outputs.~\newline
